Impact GUI Help

ImpactGUI is a graphical user interface for post processing data from one or more recording(s) of an impact test. Files should follow the documentation for impact test files that you can find in the ABRAVIBE user manual, and the file name should end with the .imptime filename extension.

Prior to Starting This GUI

Before you start up this gui, you should have at least one time recording from an impact test, where you have impacted the structure several times, with long pauses in between, to allow the responses to completely die out. How long you have to wait depends on the structure, so you need to investigate this before you start acquiring all your data. See the section Pretest – Optimize FFT Settings below.

Short summary

Here is a quick guide to how to use the impact processing software:

·         Select input file(s) to process, and an output directory, prefix, and first file number for the results files.

·         To change the Settings, select one of the files and a channel in the pretest panel, and press Start. In the Optimize FFT Settings window that opens, you can adjust the FFT settings and Export the results.

·         Select Continue in the main GUI. Select which processing mode to use, and go through each .imptime file and process it into frequency response functions etc.

Main GUI

 

Input Files

This panel allows you to select files, and reports how many files were selected. Note that all files you want to select have to be in the same directory. If you have input files in several directories, you can use the Merge Files GUI to copy files into a common directory so that you can process all .imptime files in one operation.

Output Files

In this panel you can select an output directory, a file prefix, and a start number for the first file. The FRFs etc. will be stored in files with the names <Prefix><Number>H.mat, <Prefix><Number>C.mat, and <Prefix><Number>F.mat for FRF, Coherence function, and Force spectrum, respectively, for example, with the settings according to the picture above, the results files will be named abra1H.mat, abra1C.mat, abra1F.mat, abra2H.mat, abra2C.mat, …

Pretest

The pretest panel allows you to select one of the input files, and select one out of the (possibly more than one) response channels to use for optimizing the FFT settings, see below.

Load Settings

This allows you to load a previously saved file with all settings.

Save Settings

This allows you to save all settings to a file.

Browse Files

Starts the ABRABROWSE GUI to allow you to browse and plot data.

Continue

The Continue button opens the Post Processing GUI, see below.

Copy Files GUI

 

 

The Copy Files GUI contains a list window with all files currently in the output directory, and three buttons; one to select individual files to add to the output directory, one to select one or more directories under which .imptime files are located, and one to delete files. The idea of this functionality is to gather all files you have, and copy them into one directory, in which you can then select all files for processing them together with the “Impact Processing” GUI, see below. Note that new file names may be created for your files in the output directory, as this GUI ensures that all files are copied into the output directory without overwriting existing data. You therefore need to make sure that you only copy each of your files once; otherwise duplicates will appear in the output directory.

 

Select Files

This button opens a standard file selection window, in which you can select one or more file(s) for copying into the output directory. However, you can press the button again, and select a new file to copy, if it is located in another directory than the first file(s). If you have files spread over many directories, this may be the easiest way to copy all your files.

Select Directories

This button allows you to first select a top directory by using the button to the right of the “Input Dir.” text field. The dialog that opens allows you to select a top directory under which your data is stored in subdirectories, as in the following picture.

 

 

In the picture above, you should choose “Select Folder” without selecting any of the subdirectories. It is thus, in the example showed, the directory named “DataLogger” that is selected (actually this directory could have been selected in its top directory also, by selecting the directory “DataLogger” and then pressing “Select Folder”, instead of double clicking it to get into the screen shown in the figure above). Then, the list in the Select Directory GUI is updated with all the subdirectories of the selected top directory, see figure below. Select the directories that you want to use for data merging (i.e. those directories that contain the .imptime files you want to copy), and press “Select”. ALL .imptime files located in the selected subdirectories will be copied into the output directory. If needed (if the files in the different subdirectories have the same file names, or if the files lack file numbers), the files will be renamed in the output directory. This functionality is especially valuable if you have software that stores all results files into subdirectories, for example as shown in the figure above.

 

Delete Files

To delete files in the output directory (for example because you have accidentally imported the same file twice), select the file(s) that you wish to delete in the “Current” list in the “Copy Files” GUI, and then press “Delete Files”. A warning dialog is opened, and if “OK” is chosen, the selected files will be deleted.

Pretest – Optimize FFT Settings

 

 

This GUI allows changing all FFT settings; the trigger conditions, blocksize, and window settings. It also allows you to investigate the results of the settings, to help you obtain the optimal settings to use.

 

Step 1 – Select Impacts

Press the Plot button, which produces the following plot window.

 

 

The window shows the raw time data in blue, each block defined by the trigger conditions and blocksize in green, and possible double impacts in red as in the above plot. Click on the impacts to use for FRF/Coherence calculation for the optimization. Only the selected impacts are used in the averaging process to obtain FRF and Coherence.

Step 2 – Select Triggering

After selecting a trig level and number of pretrigger samples, you can press the Plot Time button to get a plot of the result. You can zoom in and examine the data in more detail in the plot that opens. Note that after changing trig level and/or pretrigger, you need to reselect the impacts you wish to use in the “Select Impacts” panel, as trigger settings may change which blocks are detected as double impact blocks, etc.

Step 3 – Select Blocksize

This choice allows you to plot the FRFs for three blocksizes; the blocksize chosen in the popup menu, and one blocksize smaller and larger, respectively, by pressing Plot FRF. The idea with this is to investigate the bias errors at the resonance peaks in the FRFs. When the blocksize is sufficiently large, and thus the frequency increment sufficiently small, the bias error “vanishes”. So, you should try to obtain a plot like the following, in which the two larger blocksizes have approximately the same peak, which means that the bias is negligible.

 

 

Left: Full plot obtained by selecting “Plot FRFs” in the Select Blocksize panel. Right: The first resonance peak zoomed in. The blocksize 16K is optimal as it gives no bias error.

 

Note 1! After changing the blocksize setting, you may need to reselect the impacts to use for the analysis, this is then indicated by the text in the “Select Impacts” panel saying “No Impacts Selected!”. This is because the double impact detection has been affected by the selected blocksize.

Note 2! This function does not apply any force, nor any exponential window.

Note 3! If selecting a too large blocksize, you may find that some blocks overlap the next block, and are therefore detected as double impacts. If this occurs and you still have bias according to the Plot FRFs plot result, you need to measure your data again with longer intervals between the impacts!

Step 4 – Select Windows

In this panel you can adjust the force and exponential window factors and plot the results in both time domain and frequency domain by the two buttons Plot Time and Plot FRF/Coherence, respectively.

Continue GUI

The Continue button in the Main GUI opens a new GUI, and two plots as in the next figures: The plots show the results depending on the selected settings.

 

In this GUI you can process each .imptime file into FRFs, coherence functions, and force spectra. There are several different modes to process the data; see below.

Plot Channels

In this selection list you can select one or more of the response channels in the .imptime file to use for plotting FRFs and coherence functions. Since the impact hammer is on channel 1, you can only select a channel from 2 and upwards. This choice is only affecting the plots; all channels are always processed!

Process Data

In this panel you should first select the process mode. The action is then a little different depending on the Process Mode selected.

Process Mode

Freq. Manual

This choice is performing “standard” impact testing, i.e. with averaging in the frequency domain. This GUI also allows you to select which impacts to use in the Use Impacts # selection list.

Time Synchr. Manual

This choice is performing the averaging in the time domain. Note that the Coherence function is undefined in this mode.

Best Two

This choice performs an automatic processing of the files in the list starting with the Current file… and all succeeding files in the file list using an optimization algorithm. The algorithm used is the one described in paper [2], see References.

Current File

This popup menu shows which file is currently being processed. You can select any file, if you for some reason want to skip a .imptime file, or go back and reprocess a file. Note however! The output files are incremented each time you press the Save and Next button (see below), so you risk ending up with either some files unprocessed, or with multiple results for the same .imptime file!

Save and Next

This button saves the currently displayed results (including the results for the channels that are not displayed) and processes the next .imptime file in the list. If the current file is the last in the list, the GUI is closed after saving the results.

Use Impacts #

In this selection list you can select the impacts you want to use for the averaging process, if the process is one of the manual modes; otherwise this list is not shown. The default for each new .imptime file is to use all impacts except those with detected double impact. You should try to optimize the coherence by deselecting impacts which are deviating from the others. You can also sometimes note that the coherence drastically improves when you deselect a particular impact; this is an indication that the impact in question was erroneous, for example because the point of the impact was slightly off compared to the other impacts.
